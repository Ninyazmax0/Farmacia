Introducción
El presente Diccionario de Datos describe de forma detallada cada una de las tablas y sus campos que conforman la base de datos del sistema de inventario de la Farmacia Santa Cruz. Para cada campo se especifica: nombre, tipo de dato, longitud, descripción funcional y restricciones aplicadas.
Leyenda de Restricciones
PK  Llave Primaria: identifica de forma única cada fila.
FK → Tabla(campo)  Llave Foránea: referencia a otra tabla.
NOT NULL  El campo no puede estar vacío.
AUTO_INCREMENT  El valor se genera automáticamente.
UNIQUE  El valor no puede repetirse en la tabla.

Relaciones entre Tablas

1 → * Un registro puede relacionarse con muchos de la otra tabla, pero no al revés.  |  N : M Muchos a muchos, resuelto con tabla intermedia.

#Tabla OrigenCardinalidadTabla DestinoDescripción
R1Usuario1 → *CompraUn usuario registra muchas compras; cada compra la autoriza un usuario.
R2Usuario1 → *VentaUn usuario procesa muchas ventas; cada venta la atiende un usuario.
R3Proveedor1 → *CompraUn proveedor tiene muchas órdenes de compra; cada compra es de un proveedor.
R4Categoria1 → *ProductoUna categoría agrupa muchos productos; cada producto pertenece a una categoría.
R5Compra1 → *DetalleCompraUna compra tiene muchas líneas de detalle; cada detalle pertenece a una compra.
R6Producto1 → *DetalleCompraUn producto puede aparecer en muchos detalles de compra.
R7Venta1 → *DetalleVentaUna venta tiene muchas líneas de medicamentos; cada detalle pertenece a una venta.
R8Producto1 → *DetalleVentaUn producto puede venderse en muchas ocasiones distintas.
R9Producto1 → *AlertaUn producto puede generar múltiples alertas a lo largo del tiempo.
—Compra + ProductoN : MDetalleCompraUna compra incluye varios productos y un producto puede estar en varias compras.
—Venta + ProductoN : MDetalleVentaUna venta incluye varios productos y un producto puede estar en varias ventas.
